Frontend Player Launch API
Use this one backend API for all provider/game launch. Provider URL, path, payload and response path are controlled above in BO.

            

              Frontend call: POST /api/player/provider/launch

              Header: Authorization: Bearer MEMBER_JWT_TOKEN

              Body by game ID: { "gameId": 1, "amount": 10 }

              Body by provider/game code: { "providerCode":"JDB", "gameCode":"1001", "amount": 10 }

              Backend will auto create provider player if missing, optionally transfer main wallet to provider, call configured launch API, then return data.launchUrl.
